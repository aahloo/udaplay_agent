# UdaPlay - AI Game Research Agent

## Project Overview

**UdaPlay** is an AI-powered research agent for the video game industry that combines:
- **Local knowledge** via RAG with ChromaDB vector database
- **Web search capabilities** via Tavily API
- **Intelligent routing** to decide when to use internal vs. external sources
- **Self-evaluation** to assess retrieval quality
- **Stateful conversation** management with memory

This project demonstrates agentic RAG patterns where the agent autonomously decides its workflow based on tool results.

```python
import json

data_dir = "games"

for file_name in sorted(os.listdir(data_dir)):
    if not file_name.endswith(".json"):
        continue

    file_path = os.path.join(data_dir, file_name)
    with open(file_path, "r", encoding="utf-8") as f:
        game = json.load(f)

    # Format document text (customize as needed)

    # Use filename without extension as ID
    doc_id = os.path.splitext(file_name)[0]

    # Add to collection with metadata
    collection.add(
        ids=[doc_id],
        documents=[content],
        metadatas=[game]  # Preserve all game fields
    )
```

#### 3. Create EvaluationReport Pydantic Model
```python
class EvaluationReport(BaseModel):
    """Structured output for retrieval evaluation"""
    useful: bool = Field(
        description="Whether the documents are useful to answer the question"
    )
    description: str = Field(
        description="Detailed explanation of the evaluation result"
    )
```

**Purpose**: This model ensures the LLM-as-judge returns structured, parseable output.

#### 5. Tool 1: retrieve_game

```python
@tool
def retrieve_game(query: str):
    """
    Semantic search: Finds most relevant results in the vector DB

    args:
    - query: a question about game industry

    You'll receive results as list. Each element contains:
    - Platform: like Game Boy, PlayStation 5, Xbox 360...
    - Name: Name of the Game
    - YearOfRelease: Year when that game was released for that platform
    - Description: Additional details about the game
    """
    results = collection.query(
        query_texts=[query],
        n_results=5,  # Retrieve top 5 matches
        include=['documents', 'metadatas', 'distances']
    )

    # Format results for agent
    if not results['documents'][0]:
        return "No games found matching your query."

    formatted_results = []
    for i, (doc, metadata) in enumerate(zip(results['documents'][0], results['metadatas'][0])):
        game_info = (
            f"Game {i+1}:\n"
            f"  Name: {metadata['Name']}\n"
            f"  Platform: {metadata['Platform']}\n"
            f"  Year: {metadata['YearOfRelease']}\n"
            f"  Genre: {metadata.get('Genre', 'N/A')}\n"
            f"  Publisher: {metadata.get('Publisher', 'N/A')}\n"
            f"  Description: {metadata['Description']}\n"
        )
        formatted_results.append(game_info)

    return "\n".join(formatted_results)
```

#### 6. Tool 2: evaluate_retrieval

```python
@tool
def evaluate_retrieval(question: str, retrieved_docs: str):
    """
    Based on the user's question and retrieved documents, analyzes if documents
    are sufficient to answer the question.

    args:
    - question: original question from user
    - retrieved_docs: retrieved documents from Vector Database

    The result includes:
    - useful: whether the documents are useful to answer the question
    - description: detailed explanation of the evaluation result
    """
    # Initialize LLM as judge
    judge_llm = LLM(model="gpt-4o-mini", temperature=0.3)

    # Craft evaluation prompt
    evaluation_prompt = f"""Your task is to evaluate if the documents are enough to respond to the query.
Give a detailed explanation, so it's possible to take action to accept it or not.

Question: {question}

Retrieved Documents:
{retrieved_docs}

Evaluate whether these documents contain sufficient information to answer the question.
Consider:
- Do the documents directly address the question?
- Is the information specific and accurate?
- Are there any gaps in the information needed?

Provide your evaluation as JSON with 'useful' (boolean) and 'description' (string) fields."""

    # Get structured response
    response = judge_llm.invoke(
        input=evaluation_prompt,
        response_format=EvaluationReport
    )

    # Parse with Pydantic parser
    parser = PydanticOutputParser(model_class=EvaluationReport)
    try:
        evaluation = parser.parse(response)
        return f"Useful: {evaluation.useful}\nReasoning: {evaluation.description}"
    except Exception as e:
        # Fallback if parsing fails
        return f"Useful: False\nReasoning: Error evaluating retrieval - {str(e)}"
```

#### 7. Tool 3: game_web_search

```python
@tool
def game_web_search(question: str):
    """
    Search the web for gaming industry information

    args:
    - question: a question about game industry
    """
    client = TavilyClient(api_key=TAVILY_API_KEY)

    # Perform web search
    search_results = client.search(
        query=question,
        max_results=5
    )

    # Format results
    if not search_results.get('results'):
        return "No web results found."

    formatted_results = []
    for i, result in enumerate(search_results['results'], 1):
        formatted_results.append(
            f"Result {i}:\n"
            f"  Title: {result.get('title', 'N/A')}\n"
            f"  URL: {result.get('url', 'N/A')}\n"
            f"  Content: {result.get('content', 'N/A')}\n"
        )

    return "\n".join(formatted_results)
```

#### 8. Configure Agent

```python
agent = Agent(
    model_name="gpt-4o-mini",
    tools=[retrieve_game, evaluate_retrieval, game_web_search],
    temperature=0.7,
    instructions="""You are UdaPlay, an AI research agent specializing in video game industry knowledge.

Your workflow:
1. When asked a question, ALWAYS start by using retrieve_game to search your internal knowledge base
2. Use evaluate_retrieval to assess if the retrieved documents sufficiently answer the question
3. If evaluation indicates documents are useful (useful=True), provide a comprehensive answer based on them
4. If evaluation indicates documents are NOT useful (useful=False), use game_web_search to find current information
5. Always cite your sources clearly:
   - For internal knowledge: "According to my game database..."
   - For web search: "According to [source name/URL]..."
6. Be conversational and helpful, but accurate and factual
7. If you cannot find relevant information even after web search, clearly state the limitations

Remember: You have memory of conversation history, so you can reference previous exchanges.
Maintain context across multiple questions in the same session."""
)
```

```python
def print_agent_output(run, query_num, question):
    print(f"\n{'='*80}")
    print(f"QUERY {query_num}: {question}")
    print(f"{'='*80}")

    final_state = run.get_final_state()
    messages = final_state["messages"]

            for tc in msg.tool_calls:
                print(f"  - Called: {tc.function.name}")

    # Show final answer
    print("\n--- Final Answer ---")
    final_answer = messages[-1].content if messages else "No response"
    print(final_answer)
    print(f"{'='*80}\n")

# Test Query 1: Internal knowledge (should be in DB)
```

## Common Pitfalls and Solutions

### Issue 1: ChromaDB Collection Not Found
**Error**: `Collection 'udaplay' not found`

**Solution**:
- Ensure Part 1 was completed and chromadb/ folder exists
- Check collection name matches between Part 1 and Part 2
- Verify using same `path` parameter in PersistentClient

---

### Issue 2: Embedding Function Mismatch
**Error**: Queries return incorrect/empty results

**Solution**:
- Use **identical** embedding function in Part 1 (creation) and Part 2 (loading)
- If using OpenAIEmbeddingFunction, use same API key
- Don't mix embedding functions (e.g., default vs. OpenAI)

---

### Issue 3: EvaluationReport Parsing Fails
**Error**: `Pydantic validation error` or `JSON decode error`

**Solution**:
- Ensure LLM is called with `response_format=EvaluationReport`
- Verify Pydantic model has Field descriptions
- Add try/except with fallback (as shown in evaluate_retrieval example)
- Check that LLM is returning JSON format

---

### Issue 4: Agent Doesn't Use Tools Correctly
**Symptom**: Agent never calls evaluate_retrieval or game_web_search

**Solution**:
- Review agent instructions - must explicitly describe workflow
- Ensure tools are in the tools list
- Check tool docstrings are clear and descriptive
- Temperature too low (<0.3) may make agent less exploratory

---

### Issue 5: Tavily API Key Invalid
**Error**: `Unauthorized` or `Invalid API key`

**Solution**:
- Get free API key from https://tavily.com/
- Verify key is in .env file as `TAVILY_API_KEY`
- Restart kernel/notebook after adding to .env
- Check for extra spaces or quotes in .env

---

### Issue 6: Python 3.14 Compatibility
**Error**: `ConfigError: unable to infer type for attribute "chroma_server_nofile"`

**Solution**:
- Use Python 3.13 or earlier (ChromaDB not compatible with 3.14 yet)
- Create new venv with `python3.13 -m venv venv`
- Reinstall all dependencies
